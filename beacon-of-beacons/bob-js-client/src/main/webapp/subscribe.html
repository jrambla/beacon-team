Register a new beacon

If you know of a beacon which is not integrated in our system yet, please send us an email and include the beacon's URL and API details.

Beacon of Beacons also supports anonymous beacons. If you have multiple projects exposed as beacons and want to hide their identity, we can aggregate their responses under a beacon with a different name. Please specify this in your request.

If you have administrative control over your beacon, please make sure it supports the following API:

  • The query is an HTTP GET request of the following format: http://[BEACON-URL]?chrom={chromosome}&pos={position}&allele={allele}&ref={reference}, where {chromosome} is the chromosome ID (one of the 1-22, X, Y, MT), {position} is a 1-based coordinate within a chromosome, {allele} is any string of nucleotides A,C,T,G or D, I for deletion (insertion) and {reference} is the genome ID prefixed with "hg" (e.g. hg38, hg19, hg18). The first three parameters are compulsory, the last one is only required if the beacon supports multiple genomes. The order of the parameters does not matter and their values are case-insensitive. These parameters should be enough to query the beacon, although further options may be recognized as well (e.g. to request a response in a different format).
  • The response is a plaintext string with one of the following values: YES (the variant has been found), NO (the variant has NOT been found) and NULL (an error occured, e.g. the request is not valid).

For more information about beacons, please visit the web of the Beacon Project.
